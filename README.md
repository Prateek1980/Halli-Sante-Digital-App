# Halli Santhe Digital App

## Overview

Halli Santhe Digital App is an Android-based hyper-local marketplace application designed to connect local artisans and small-scale sellers directly with customers. The platform helps promote handmade and local products by providing an accessible digital marketplace experience.

This project was developed as part of an academic initiative focused on solving real-world accessibility and marketplace challenges faced by local sellers.

---

## Problem Statement

Local artisans and small sellers often struggle to reach a wider customer base due to:

- Limited digital presence
- Lack of affordable online marketplace access
- Dependence on traditional selling methods

Customers also face difficulty discovering authentic handmade and local products in a single convenient platform. This creates a gap between artisans and potential buyers, limiting business growth and accessibility.

---

## Proposed Solution

To address these challenges, the application provides:

- A mobile-based digital marketplace
- Product categorization for easy browsing
- User authentication system
- Product search functionality
- Detailed product display pages
- Direct customer communication options

The platform simplifies product discovery while improving digital visibility for local sellers.

---

## Features

- User Registration and Login
- Product Categories
- Product Search
- Product Detail View
- Seller Contact / Inquiry Option
- Clean and User-Friendly UI
- Firebase Authentication Integration
- Firebase Database Integration

---

## Technologies Used

Used for Android app development XML – Used for designing the user interface layouts Android Studio – Primary development environment for building and testing the application Firebase Authentication – For user signup and login functionality using email/password authentication Firebase Realtime Database / Firestore – For storing user and product-related data Android SDK – For app functionalities, UI components, and device compatibility GenAI / AI-assisted Development Tools – Used for development assistance, code suggestions, and faster implementation Git / Version Control – For project tracking and code management

## Project Structure

The project follows standard Android application architecture using:

- Activities
- XML Layouts
- Firebase Services
- Recycler Views
- Kotlin-based Android Components

## Final Conclusion

The project successfully created a functional prototype of a hyper-local artisan marketplace that helps bridge the gap between local sellers and customers.
